| 1. 서비스 정의 | 1) T앱 & PC웹 내에서 퀵 서비스를 신청 / 전화접수 (1833-8153 / 비즈 어드민에서 퀵 접수 가능 기업으로 등록된 기업만 해당) | | | 4. 주요 기능 | 1) 주문서 작성 | | | |
| - 타 버티컬과 중복으로, 퀵 버티컬 내 중복으로 신청 가능 | | | - 주문서: T앱/PC웹에서 배송 서비스 접수에 필요한 정보를 입력, 결제까지의 단위 | | | |
| 2) 퀵 서비스는 퀵 배송과 택배 배송으로 구성 | | | - 고객 | | | |
| 3) 서비스에서 수행하는 퀵 배송은 소화물 & 대형 | | | 일반: T\_Uid가 정상 상태인 유저가 T앱을 통하여 주문 진행 | | | |
| - 중박스(가로 세로 높이의 합 140cm & 20kg이하)까지 소화물로 정의 | | | 비즈니스 간편가입: T앱을 이용하여 주문 진행 | | | |
| 2. 유저 정의 | 1) 고객(주문자): 주문이 발생한 T\_Uid 고객 (주문부터 결제까지 진행한 고객) | | | 비즈니스 기업가입: T\_Uid가 ‘퀵’ 버티컬 사용 Y인 경우 T앱/PC웹을 통해 주문 진행 | | | |
| 2) 고객(발신자): 발신자로 기재된 출발지 고객 | | | - 1개의 주문에 매핑되는 주문정보(출/도착, 물품, 차량, 결제)는 단건 | | | |
| 3) 고객(수신자): 수신자로 기재된 도착지 고객 | | | 2) 주문내역 조회 | | | |
| 4) 퀵 픽커(기사): 퀵 주문 처리 주체 | | | - 주문내역: 고객이 결제를 진행한 주문 건에 대한 내역 조회 단위 | | | |
| 5) 택배기사: 택배 주문 처리 주체 | | | - 고객은 진행중인 주문의 정보 조회 가능/취소 가능 | | | |
| 3. 상품 유형 | 1) 퀵급송: 픽커앱 기사를 통한 급송 배송 (요금 : 일반대비 b2b 1.7배 / b2c 1.7 배) B2B 기업마다 일부 차이 있음(하이마트 1.2배) | | | - 고객은 진행 완료된 주문의 내역 조회 가능 | | | |
| 2) 퀵: 픽커앱 기사를 통한 일반 배송 | | | 5. 계정 정책 | 1) 고객 계정은 카카오 T user 계정이며, 카카오 T user의 공통 정책을 따름 | | | |
| 3) 안전배송: 승용차 fleet의 픽커를 통한 일반 배송 (요금 : 일반대비 110% 수준) | | | 6. 결제 정책 | - 퀵 : 기업가입 (Y: 카드 N: 카드, 카드+포인트, 휴대폰), 간편가입&일반 (카드, 카드+포인트, 휴대폰) | | | |
| 4) 퀵 이코노미: 픽커앱 기사를 통한 느리지만 저렴한 배송 (21/12/8 추가) (요금 : 일반대비 70% 수준) | | | - 택배 : 착불 없음 외 퀵과 상동 | | | |
| 5) 퀵 대형: 다마스, 라보, 1톤을 통한 대형 물품 배송 (급송, 이코노미 등 없음) | | | 7. 배송요청사항 | 글자수 제한 : T앱 / PC웹 100자 | | | |
| 6) 택배: 제휴 택배사(한진/한진 운영 이슈상 불가 시 CJ로 전환) 공급망을 통한 배송 | | |  |  | | | |
